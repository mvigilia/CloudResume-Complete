    How this Website Works?

    This website is built under AWS Services, utilizing S3 for the website hosting with HTML, CSS and JavaScript for the front end. While the backend infrastructure is supported by multiple AWS Services, namely API Gateway, Lambda, DynamoDB, and Simple Notification Service (SNS).

    • Route 53: A highle scalable DNS web service which routes user requests to endpoints,applications, or other AWS Services. Translating domain names to IP Address that computers use to identify each other on the network.

    • ACM Manager: AWS service that helps you provision, manage, and deploy Secure Sockets Layer (SSL) and Transport Layer Security (TLS) certificates for use with AWS services.

    • S3: AWS service object storage service that allows you to store and retrieve any amount of data, such as files, images, videos, or backups, with high durability and scalibility. It is designed for easy access, enabling you to store data with a simple web interface.

    • Lambda: A serverless compute service that allows you to run code in response to events (such as HTTP request, database changes, or file uploads) without provisioning or managing serves. Lambda automatically scales your code exectuion and you only pay for the compute time you use.

    • DynamoDB: A fully managed NoSQL database service that provides fast and predictable performance with seamless scalability. It is ideal for applications that require low-latency data access, such as real-time annalytics, mobile apps, and e-commerce platforms.

    • Simple Notification Service (SNS) : A fully managed messaging service that allows you to send notifications to users or systems via multiple communication channels, such as e-mail, SMS, or mobile push notifications. It can also be trigger automated actions, such as invoking AWS lambda functions or sending messages to other AWS services, making it ideal for alerting, messaging, and event-driven workflows.
